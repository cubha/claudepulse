## 1. 리서치 배경

기존 team-research에서 도출된 게임 주제(삼겹살 뒤집기, 회사 야근 달리기 등)가 **실제 사용 맥락과 맞지 않아** 재조정이 필요하다.

**실제 사용 맥락**: 친구들끼리 볼링, 야구 등을 하고 **진 팀에서 1명이 돈을 전부 내는 걸 정하는 게임**. 회식/직장 문화가 아닌, 누구나 아는 대중적인 게임으로 "누가 당첨되는지"를 재미있게 보여주는 것이 핵심.

**기존 리서치에서 제거할 내용**:
- 회식/직장문화 테마 (삼겹살, 야근, 부장님 카톡 등)
- 아이템 편향지급 (마리오카트 블루쉘 패턴) — 이 게임은 조작 없이 순수 관람형
- 규칙반전 ("꼴등이 1등" 선언) — 직관성 훼손
- 직접 조작 게임 (연타, 클릭 타이밍) — 관람형이 맞음

**유지할 내용**:
- 기술 스택 권장안 (PixiJS + Matter.js + GSAP + Howler.js + Vite)
- 물리 불확실성 메커니즘
- 30초 게임 루프 기본 구조
- Late-Game 카오스 주입 (물리 기반으로만)

## 2. 조사 결과

### 2-1. 대중적 게임 주제 TOP 5 (재선정)

모두가 알고, 규칙 설명 없이 바로 이해할 수 있는 게임만 선별했다.

#### TOP 1. 경마 (Horse Racing) — 추천도: ★★★★★

> 참가자 이름이 적힌 말들이 트랙을 달린다. 물리+랜덤으로 순위가 결정된다.

| 항목 | 내용 |
|---|---|
| **인지도** | 최상 — 경마는 전 세계 공통 게임, 별도 설명 불필요 |
| **30초 적합성** | 매우 높음 — 직선/곡선 트랙, 속도 변화로 30초 채우기 용이 |
| **관전 재미** | 매우 높음 — 접전, 역전, 추월이 자연스럽게 발생 |
| **구현 난이도** | 낮음 — 횡스크롤 레이스, 물리엔진 필요 없이 랜덤 속도 변화만으로 가능 |
| **역전 가능성** | 높음 — 직선 구간에서의 스퍼트, 코너링 감속 등 자연스러운 역전 |
| **참고 사례** | [Horse Race Timer](https://horseracinggame.net/simulator) — 2~100마리, 무료, 로그인 불필요, 파티 최적화, [Track Night](https://www.track-night.com/) — QR코드 폰 참가 + TV 연결, 오프라인 파티 최적, [Photo Finish Racing](https://photofinishracing.com/) — 포토피니시 연출이 핵심인 보드게임, [Streamer.bot Horse Race](https://extensions.streamer.bot/t/horse-race-mini-game/177) — 트위치 스트리머용 경마 미니게임 |
| **왜 1위인가** | 가장 보편적, 관전 재미 최고, "내 말이 달린다"는 몰입감, 접전 연출이 자연스러움 |

#### TOP 2. 구슬 레이스 (Marble Race) — 추천도: ★★★★★

> 참가자 이름이 적힌 구슬들이 물리 트랙을 굴러 내려간다.

| 항목 | 내용 |
|---|---|
| **인지도** | 높음 — Jelle's Marble Runs (유튜브 200만+ 구독), lazygyu 마블 룰렛 |
| **30초 적합성** | 매우 높음 — 트랙 길이로 시간 조절 가능 |
| **관전 재미** | 최상 — 물리 충돌로 예측 불가능, 순위 수시 변동 |
| **구현 난이도** | 중 — Matter.js 물리엔진 활용, 트랙 설계 필요 |
| **역전 가능성** | 최상 — 충돌/반동으로 꼴등이 1등 되는 장면이 자연 발생 |
| **참고 사례** | [Jelle's Marble Runs](https://jellesmarbleruns.com/) — 물리 기반 구슬 레이스의 대명사 (유튜브 200만+), [lazygyu 마블 룰렛](https://lazygyu.github.io/roulette/) — Box2D WASM 기반 웹 구슬 레이스, [Marble Roulette](https://namu.wiki/w/Marble%20Roulette) — 나무위키 독립 문서 존재, 치지직/SOOP 방송 추첨에 실사용 중, [Gravity Picker](https://www.gravitypicker.com/) — 물리 기반 이름 뽑기 + 속도 조절 + 결과 영상 다운로드, [Marble Race (Evolution Gaming)](https://games.evolution.com/live-casino/game-shows/marble-race/) — 카지노 베팅 게임으로 정식 출시된 구슬 레이스 |
| **왜 2위인가** | 물리 기반 예측불가능성이 가장 극적. Jelle's가 증명한 "순수 물리 레이스의 관전 재미". 한국에서도 Marble Roulette이 나무위키 독립 문서가 생길 정도로 인지도 높음. Evolution Gaming이 카지노 베팅 게임으로 정식 출시할 정도로 검증된 포맷 |

> **Jelle's Marble Runs 성공 요인**: "관객도 제작자도 결과를 모른다" — 순수 물리/중력에 의한 랜덤 결과가 조작된 스크립트보다 더 극적인 서사를 만든다. 실황 중계 스타일의 해설이 구슬에 인격을 부여하여 감정 이입을 유도. ([PC Gamer](https://www.pcgamer.com/get-ready-for-marbles-to-be-the-next-big-esport/), [Phantom Knight Games](http://www.phantomknightgames.com/news/jelles-marble-runs-blaseball-and-the-joy-of-chaos/))

> **Gravity Picker 참고 포인트**: 물리 기반 이름 뽑기 + 속도 조절 + **결과 영상 다운로드** 기능이 SNS 확산에 직접 기여. 스트리머용 그린스크린 모드도 제공.

## 3. 재미요소 리서치 (아이템/규칙반전 제외)

기존 리서치에서 "아이템 편향지급"과 "규칙반전"을 제거하고, **순수 관람형 게임에 맞는 재미요소**를 재조사했다.

### 3-1. 물리 기반 예측불가능성 (유지 + 강화)

기존 리서치의 "물리 불확실성"을 그대로 유지하되, 관람형에 맞게 조정.

| 요소 | 설명 | 적용 게임 | 효과 |
|---|---|---|---|
| **충돌 반동** | 캐릭터/구슬 간 충돌 시 튕겨나감 | 구슬, 핀볼 | 순위 급변, 예측 불가 |
| **랜덤 장애물** | 트랙에 갑자기 등장하는 장애물 | 경마, 구슬 | 선두 방해, 역전 유발 |
| **지형 변화** | 오르막/내리막, 좁은 구간 | 경마, 구슬 | 밀집→충돌 유발 |
| **바람/조류** | 랜덤 방향의 외부 힘 | 경마, 보트 | 전체 순위 교란 |

> **핵심**: 플레이어 개입 없이 **물리 법칙 자체가 드라마를 만드는** 것이 이 게임의 본질. Jelle's Marble Runs가 증명한 패턴.

### 3-2. 시각/청각 연출 (신규)

관전의 재미를 극대화하는 연출 기법. 실제 스포츠 중계에서 차용.

| 요소 | 설명 | 적용 시점 | 근거 |
|---|---|---|---|
| **실황 중계풍 자막** | "OO가 추월한다!", "접전이다!" 등 자동 생성 자막 | 순위 변동 시 | Jelle's Marble Runs의 해설이 구슬에 인격 부여 → 감정이입 유도 |
| **포토피니시 슬로우모션** | 결승선 통과 순간 0.3x 슬로우모션 + 카메라 줌인 | 마지막 1~2초 | Photo Finish Racing 보드게임 — 접전 연출이 게임의 핵심 매력 |
| **순위 변동 알림** | 순위가 바뀔 때 화면 효과 + 효과음 | 추월 발생 시 | 슬롯머신 근거 — "near win" 이펙트가 흥분 유발 |
| **BGM 가속** | 마지막 5초 BGM 템포 1.5배 + 심장박동 효과음 | 25~30초 | 긴장감 생리적 각성 유도 |
| **카메라 팔로우** | 1등과 꼴등을 번갈아 비추는 카메라 | 레이스 중반 | 스포츠 중계 기법 — 선두/추격자 교차 편집 |
| **역전 시 화면 흔들림** | 순위 큰 변동 시 화면 shake + 폭발 이펙트 | 3위 이상 변동 시 | 시각적 충격으로 관전 흥미 극대화 |

### 3-3. 결과 발표 연출 (신규)

게임 끝난 후의 연출이 SNS 공유와 재플레이 욕구를 좌우한다.

| 요소 | 설명 | 효과 |
|---|---|---|
| **1등 시상식** | 황금 폭죽 + 왕관 + 팡파레 + "축하합니다!" | 1등의 우월감 극대화 |
| **꼴등 벌칙 연출** | 낙뢰 + 슬픈 트롬본 + 검은 배경 + "당신이 쏩니다" | 꼴등의 비극적 유머 (핵심 — 이 사람이 돈 냄) |
| **전체 순위 리플레이** | 3초 압축 리플레이 (핵심 역전 장면 하이라이트) | SNS 공유 소재 |
| **결과 카드** | "OO님이 꼴등! 오늘 N만원 쏩니다" 이미지 생성 | 카카오톡/인스타 공유용 |
| **리플레이 버튼** | "한 판 더?" — 즉시 재시작 | 재플레이 유도 |

### 3-4. 긴장감 조성 기법 (기존 리서치 보완 + 에이전트 추가 조사)

**게임 디자이너 Chris Pruett의 "The Mechanics of Tension" 연구 기반 검증된 패턴:**

| 기법 | 설명 | 게임 설계 적용 |
|---|---|---|
| **리듬의 교차** | 빠른 레이스 → 슬로우모션 → 갑작스러운 결과 확정. "고조-이완-폭발" 리듬 | 레이스 중반 속도감 → 결승 직전 슬로우 → 결과 팡파르 |
| **불확실성 지속** | 결승선 직전까지 1위가 바뀌도록 설계. 50% 이후에도 역전 가능해야 집중 | 전반: 속도 변동 크게 / 후반: 작게 → 마지막 이벤트로 역전 |
| **지연된 결과** | 결과가 바로 나오지 않고 슬로우모션-줌인-팡파르 순서로 3~4초 지연 | 인간의 뇌는 답을 기다리는 시간 자체에서 도파민 분비 |
| **공개 선언** | "OO이 꼴등!" 이름이 화면에 크게 표시되는 순간 → 집단의 유머 포인트 | 결과 화면에 이름 대문짝만하게 + 벌칙 문구 |
| **근사 순위 표시** | 정확한 거리 대신 "접전/선두/추격중" 정도만 표시 | 마지막 3초에 정확 순위 공개 → 서프라이즈 |
| **밀집 구간 설계** | 트랙 중간에 좁은 구간 배치 → 전원 밀집 → 충돌 → 순위 리셋 | 구슬/경마 트랙 설계 시 의도적 병목 |
| **카운트다운 압력** | 시작 전 3-2-1 카운트다운 자체가 긴장감 생성 | 일제히 화면 주목 → 시작 순간 환호 유도 |

### 3-5. 사회적 재미 요소 (신규)

"누가 돈 내냐" 게임의 본질은 **함께 보며 떠드는 것**. 이 맥락에 맞는 재미 요소.

| 요소 | 설명 | 효과 |
|---|---|---|
| **베팅 시스템** | 게임 전 "누가 꼴등일까?" 예측 투표 | 관전 몰입도 상승, 맞추면 추가 재미 |
| **응원 이펙트** | 터치/클릭 시 해당 캐릭터에 응원 이펙트 (게임 결과에 영향 없음) | 관전 참여감 부여 |
| **닉네임 커스텀** | 참가자 이름 입력 → 캐릭터 위에 표시 | 감정이입, "내 캐릭터" 의식 |
| **연속 기록** | "OO님 3연속 꼴등!" 통계 표시 | 웃음 유발, 재플레이 동기 |
| **명예의 전당** | 역대 꼴등/1등 기록 (로컬 저장) | 장기 사용 유도 |

## 4. 권장안

### 1순위: 경마 (Horse Racing)

**근거:**
- 인지도 최상 — 규칙 설명 불필요
- 관전 재미 최고 — 추월, 접전, 스퍼트가 자연스럽게 발생
- 구현 난이도 낮음 — 물리엔진 없이도 구현 가능 (랜덤 속도 변화)
- 포토피니시 연출이 "누가 꼴등이냐"와 완벽히 맞음
- SNS 바이럴 — "내 말이 꼴등했다" 캡처 공유

**제안 구현 방식:**
- 횡스크롤 트랙, 각 말에 참가자 이름 표시
- 속도 = 기본속도 + 랜덤 가속/감속 (프레임마다)
- 트랙 중간에 "진흙탕 구간" (전체 감속) → 밀집 → 접전 유도
- 마지막 스퍼트 구간에서 랜덤 대역전 가능
- 포토피니시 슬로우모션으로 마무리

### 2순위: 구슬 레이스 (Marble Race)

**근거:**
- 물리 기반 역전이 가장 극적 (Jelle's Marble Runs 검증)
- lazygyu 마블 룰렛이라는 검증된 레퍼런스 존재
- 트랙 디자인으로 게임 다양성 확보 가능

**경마 대비 트레이드오프:**
- (+) 물리 기반 역전의 극적 효과가 더 강함
- (-) 구현 난이도 높음 (Matter.js 물리엔진 필요)
- (-) "구슬 레이스"라는 개념이 경마보다 직관적이지 않음

### 멀티 모드 제안

하나만 고를 필요 없이, **여러 게임을 모드로 제공**하는 것도 가능:

```
1등꼴등 게임
├── 경마 모드 (기본)
├── 구슬 레이스 모드
├── 사다리타기 모드 (빠른 추첨용)
└── 핀볼 모드 (추가 확장)
```

### 주의사항

| 항목 | 내용 |
|---|---|
| **관람형 유지** | 이 게임은 플레이어가 직접 조작하지 않음. 이름 입력 → 시작 → 관람 → 결과 |
| **공정성 인식** | 조작 의심을 받으면 안 됨. 시드값 표시 또는 "물리 시뮬레이션" 강조 |
| **모바일 우선** | 친구들 모여서 폰으로 보는 상황 → 반응형 필수, 세로 모드 지원 |
| **빠른 재시작** | 한 판 끝나면 바로 다시 할 수 있어야 함 |
