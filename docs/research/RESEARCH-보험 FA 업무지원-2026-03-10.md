## 1. 프로젝트 개요

- **목적**: GA(보험대리점) 소속 보험설계사(FA)를 위한 통합 업무지원 하이브리드 웹앱 개발
- **타겟 사용자**:
  - **개인 초보 설계사** → 보험 영업 및 업무 전반 가이드/교육/길라잡이
  - **관리자(팀장/지점장급)** → 소속 직원 관리 + 개인 업무 비서 기능
- **핵심 요구사항**:
  - 준법/컴플라이언스 교육 시스템 (15개 교육 카테고리)
  - 초보 설계사 온보딩 및 단계별 학습
  - 관리자용 실적/교육 대시보드
  - 뉴스/공시 정보 자동 수집 피드
  - 영업 보조 도구 (보장분석, CRM, 전자서명 연동)
  - AI 챗봇 (규제/상품/영업 질의응답)

## 3. 아키텍처 패턴 리서치 (architecture-researcher)

### 3-1. 확정 아키텍처: Modular Monolith (NestJS + Docker)

```
┌─────────────────────────────────────────────────────────┐
│              NestJS API Server (Docker)                   │
│                    API Gateway / BFF                      │
├──────────┬──────────┬──────────┬──────────┬─────────────┤
│  Auth    │   LMS    │   CRM    │Compliance│  NewsFeed   │
│  /RBAC   │          │          │  Guide   │             │
│  Module  │  Module  │  Module  │  Module  │   Module    │
├──────────┴──────────┴──────────┴──────────┴─────────────┤
│              Shared Kernel (공통 도메인 로직)              │
├──────────┬──────────────────────────────┬───────────────┤
│ Socket.io│   PostgreSQL 16+ (Docker)    │   AWS S3      │
│ Gateway  │   + RLS Multi-Tenancy        │   + CloudFront│
│ (실시간)  │   + pgvector (RAG 임베딩)     │   (파일저장)   │
└──────────┴──────────────────────────────┴───────────────┘
```

**선택 근거**:
- MVP 단계에서 마이크로서비스는 과도한 복잡성 유발
- NestJS 모듈 시스템이 Modular Monolith 경계와 1:1 대응
- 모듈 경계가 명확하면 향후 Docker 컨테이너 단위 마이크로서비스 전환 용이
- 단일 DB로 트랜잭션 일관성 확보 (금융 데이터 무결성)
- PostgreSQL RLS가 모듈 간 데이터 격리를 자연스럽게 지원
- 자체 호스팅으로 국내 클라우드 배포 → 개인정보보호법 준수

### 3-2. 5대 핵심 모듈

| 모듈 | 책임 | 주요 엔티티 |
|---|---|---|
| **Auth/RBAC** | 인증, 4단계 역할 관리, 세션 | User, Role, Permission, Session |
| **LMS** | 교육 콘텐츠, 퀴즈, 학습 진도, 자격증 | Course, Module, Lesson, Quiz, Progress, Certificate |
| **CRM** | 고객 관리, 상담 기록, 계약 추적, 이벤트 알림 | Customer, Consultation, Contract, Event |
| **ComplianceGuide** | 준법 가이드/위키, 자가진단, 광고 심의, 제재 이력 | Article, Checklist, AdReview, Violation |
| **NewsFeed** | 규제 뉴스, 공시 수집, 사내 공지, 알림 | NewsItem, Announcement, Notification |

### 3-3. RBAC 설계 (4단계)

```
본사관리자(Admin)
  └── 지점장(BranchManager)
       └── 팀장(TeamLeader)
            └── FA(Agent)
```

| 역할 | 권한 범위 |
|---|---|
| FA | 본인 데이터 CRUD, 교육 수강, 준법 가이드 조회, CRM(본인 고객) |
| 팀장 | FA 권한 + 소속 FA 실적/교육 조회, 팀 대시보드 |
| 지점장 | 팀장 권한 + 지점 전체 관리, 민원/제재 이력 관리 |
| 본사관리자 | 전체 데이터 접근, 콘텐츠 관리, 시스템 설정 |

### 3-4. 데이터 모델 핵심 설계

```sql
-- RLS 정책 예시 (자체 호스팅 PostgreSQL)
-- NestJS에서 SET LOCAL role / SET LOCAL app.current_user_id 로 컨텍스트 주입
CREATE POLICY "fa_own_data" ON customers
  FOR ALL USING (current_setting('app.current_user_id')::uuid = fa_id);

CREATE POLICY "team_leader_view" ON customers
  FOR SELECT USING (
    fa_id IN (
      SELECT id FROM users
      WHERE team_id = current_setting('app.current_team_id')::uuid
    )
  );

-- Audit Trail (Append-Only)
CREATE TABLE audit_log (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  action TEXT NOT NULL,
  entity_type TEXT NOT NULL,
  entity_id UUID,
  payload JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);
-- INSERT ONLY, no UPDATE/DELETE policy
-- NestJS AuditInterceptor에서 자동 기록
```

### 3-5. 아키텍처 진화 로드맵

| Phase | 아키텍처 | 특징 |
|---|---|---|
| **Phase 1 (MVP)** | NestJS Modular Monolith + Docker Compose | 단일 배포, RLS 멀티테넌시, S3 파일저장 |
| **Phase 2 (성장)** | + Redis 캐시 + CloudFront CDN + CQRS 읽기모델 | 대시보드 성능 최적화, 관리자 리포트 분리 |
| **Phase 3 (확장)** | Docker 컨테이너 단위 마이크로서비스 분리 (ECS/EKS) | LMS/CRM 독립 스케일링, 이벤트 버스(Bull/BullMQ) 도입 |

### 3-6. 보안 아키텍처 (금융권 요구사항)

| 영역 | 적용 기술 | 비고 |
|---|---|---|
| **전송 보안** | TLS 1.3 + Certificate Pinning | 모바일 앱 필수 |
| **인증** | Passport.js + JWT + Refresh Token Rotation | NestJS AuthModule 자체 구현 |
| **데이터 암호화** | AES-256 (민감 필드), At-Rest 암호화 | 고객 개인정보, 계약 정보 |
| **접근 제어** | PostgreSQL RLS + NestJS Guards (RBAC) + IP 화이트리스트 | DB 레벨 + 앱 레벨 이중 방어 |
| **감사 추적** | Append-Only Audit Log (NestJS Interceptor) | 모든 데이터 변경 자동 기록 |
| **개인정보** | 개인정보 암호화, 파기 자동화, 동의 관리 | 국내 서버 호스팅으로 개인정보보호법 준수 |
| **파일 보안** | S3 Bucket Policy + Pre-signed URL | 교육 콘텐츠 접근 제어 |

### 4-5. 시장 갭 분석

**핵심 발견: 준법 교육 + 영업 도구 + 관리자 대시보드를 통합한 서비스가 없다**

| 시장 갭 | 상세 | 기회 |
|---|---|---|
| **교육-영업 단절** | 교육 앱(보험연수원)과 영업 앱(시그널플로우)이 분리 | 교육 후 즉시 실무 적용 동선 설계 |
| **GA 전용 컴플라이언스 도구 부재** | 기존 컴플라이언스는 보험사 내부 시스템 | GA 맞춤형 준법 가이드 + 자가진단 |
| **관리자 통합 대시보드 부재** | 실적/교육/민원을 각각 다른 시스템에서 관리 | 원스톱 관리자 대시보드 |
| **AI 활용 초기 단계** | 다자비만 AI 검색 제공, 교육용 AI 없음 | RAG 기반 규제 Q&A + AI 학습 추천 |
| **초보 설계사 온보딩 전용 도구 없음** | 신인 FA의 이탈률 높음 (1년 내 60%+) | 단계별 온보딩 프로그램 + 멘토 매칭 |

### 4-6. 차별화 전략 요약

1. **"교육 → 실무" 원스톱 동선**: 준법 교육 수강 후 바로 체크리스트 적용 → 계약 체결
2. **AI 준법 비서**: RAG 기반 실시간 규제 Q&A (피플라이프 교육자료 + 법령 임베딩)
3. **GA 맞춤 컴플라이언스**: 피플라이프 양정규정, 파인즈 프로세스 등 GA 내부 규정 반영
4. **관리자 통합 뷰**: 실적 + 교육 이수 + 민원/제재 이력을 하나의 대시보드에서
5. **게이미피케이션 교육**: 배지, 랭킹, 연속 학습 스트릭으로 학습 동기 부여

## 7. 기술 스택 종합 권장안

### 크로스 분석 결과

- tech-researcher의 React Native(Expo) + NestJS 확정 ↔ architecture-researcher의 Modular Monolith + PostgreSQL RLS → **완전 호환**
- market-researcher의 "AI 챗봇 차별화" ↔ tech-researcher의 Claude API + pgvector RAG → **직접 지원**
- market-researcher의 "교육+영업 통합" ↔ architecture-researcher의 5모듈 설계 → **NestJS 모듈로 1:1 구현**
- Docker 컨테이너화 → Phase 3 마이크로서비스 전환 시 모듈별 분리 자연스러움

| **AI 챗봇** | Claude API + PostgreSQL pgvector RAG | 한국어 최강, 보험 도메인 RAG |
| **실시간** | Socket.io (NestJS Gateway) | 알림, 대시보드 실시간 업데이트 |
| **파일 저장** | AWS S3 + CloudFront CDN | 대용량 교육 콘텐츠, Pre-signed URL 접근 제어 |
| **상태관리** | Zustand | 경량, DevTools 지원, 러닝커브 낮음 |
| **푸시 알림** | expo-notifications + FCM | 네이티브 푸시 알림 |
| **캐시** | Redis (Docker) | 세션, 토큰, API 응답 캐싱, Bull 큐 |
| **컨테이너** | Docker Compose (개발/스테이징) → AWS ECS (운영) | 환경 일관성, 스케일링 |
| **모니터링** | Sentry + PostHog | 에러 트래킹 + 사용자 행동 분석 |
| **CI/CD** | GitHub Actions + EAS Build | Expo 앱 빌드 + Docker 배포 자동화 |
| **테스트** | Vitest + RNTL + Supertest (API) + Jest (E2E) | TypeScript 네이티브, API 통합 테스트 |

### 확정 아키텍처 다이어그램

```
[모바일 앱: React Native/Expo]  ←→  [NestJS API Server (Docker)]
[웹 관리자: Next.js 15]        ←→    ├── Auth Module (Passport.js + JWT)
                                      ├── LMS Module
                                      ├── CRM Module
                                      ├── Compliance Module
                                      ├── NewsFeed Module
                                      ├── Socket.io Gateway (실시간)
                                      └── Shared Kernel
                                            ↕
                               ┌──────────────────────────┐
                               │ PostgreSQL 16+ (Docker)   │
                               │  + RLS + pgvector         │
                               ├──────────────────────────┤
                               │ Redis (Docker)            │
                               │  + 세션/캐시/Bull 큐       │
                               ├──────────────────────────┤
                               │ AWS S3 + CloudFront       │
                               │  + 교육 콘텐츠/파일 저장    │

                                            ↕

```

### 리스크 및 주의사항

| 리스크 | 영향 | 대응 |
|---|---|---|
| 자체 인프라 관리 부담 | DevOps 역량 필요 | Docker Compose로 로컬 통합, CI/CD 자동화, IaC(Terraform) 도입 |
| React Native 대규모 리스트 성능 | 대시보드 리스트 렌더링 | FlashList + 가상화, CQRS 읽기모델 |
| 보험사 API 연동 제한 | 보장분석/전자청약 연동 어려움 | Phase 1에서 딥링크로 대체, Phase 3에서 API 협의 |
| 금융권 개인정보 규제 | ISMS-P 인증 필요 가능성 | 국내 서버 호스팅 + 암호화 + 감사 로그 설계, 법률 자문 |
| Claude API 비용 증가 | 사용자 증가 시 AI 비용 | Redis 캐시 전략, FAQ 사전 답변 캐싱, Rate Limiting |
| PostgreSQL 운영 | 백업/복구/스케일링 자체 관리 | Docker 볼륨 + 자동 백업 스크립트, AWS RDS 전환 옵션 |
