## 1. 채택 스택 (Phase 2 RESEARCH 권장 1순위 그대로)

| 카테고리 | 선택 | 근거 |
|---|---|---|
| **언어** | TypeScript 5.x | VS Code Extension 표준 |
| **번들러** | esbuild | VS Code 공식 권장, 빌드 10-100배 빠름 |
| **타입 체크** | tsc --noEmit (별도 step) | esbuild는 트랜스파일만 |
| **VS Code API** | engines.vscode `^1.85.0` | WebviewViewProvider · StatusBarItem · WebviewPanel |
| **파일 와처** | chokidar `^4.0` | `vscode.FileSystemWatcher`는 워크스페이스 외부 감시 불가 (HARD) |
| **차트** | Chart.js `^4.4` | ~60KB, CSP 안전, 다크모드 내장, 외부 fetch 없음 |
| **메시지 패싱** | `vscode-messenger` (TypeFox) | 타입 안전 RPC, postMessage 추상화 |
| **저장** | `globalStorageUri` + JSON 인덱스 | zero native deps. 10만+ 레코드 시 SQLite 마이그 |
| **가격 데이터** | LiteLLM 가격 스냅샷 임베드 | 빌드 타임 import, 오프라인 USD 환산 |
| **테스트** | `@vscode/test-electron` + vitest | Extension Host 통합 + 유닛 |
| **린트** | ESLint + Prettier | TypeScript 표준 |
| **배포** | `vsce` + `ovsx` | VS Marketplace + Open VSX 동시 (Cursor용 필수) |

### 거부된 대안 + 거부 사유

- ❌ webpack — esbuild 대비 느림, VS Code 신규 템플릿에서 제외
- ❌ Recharts/Tremor — React 런타임(~150KB+) 부담, CSP 위배 위험
- ❌ better-sqlite3 (MVP) — 플랫폼별 native rebuild 부담, MVP 단계 데이터 양에 과잉
- ❌ DuckDB-WASM — ~8MB 번들 부담
- ❌ Anthropic Admin Usage/Cost API — Pro/Max 구독자에게 적용 불가, V2+ 보조 옵션
- ❌ Claude.ai 비공개 HTTP API — Claudemeter처럼 즉시 중단 위험 (CRITICAL 금지)

## 9. ⚠️ 구현 시 절대 지킬 것 (CRITICAL — 코드 주석에도 명시 예정)

1. **`message.id`로 dedup** — 누락 시 billing 불일치
2. **`vscode.FileSystemWatcher` 사용 금지** — 외부 경로 감시 불가, chokidar 필수
3. **`unsafe-eval` 의존 금지** — Webview CSP 위배
4. **claude.ai 비공개 HTTP API 절대 사용 금지** — 즉시 중단 위험
5. **하드코딩 색상 0개** — `var(--vscode-*)` 또는 `var(--c-*)` 토큰만
6. **5+1 액센트 cap** — 새 색 추가 시 합의 필수
